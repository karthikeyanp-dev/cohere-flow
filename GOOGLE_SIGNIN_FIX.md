# Google Sign-In Fix for Firebase App Hosting (Cloud Run) Deployment

## Problem Summary

Google sign-in fails when the application is deployed to Firebase App Hosting (Cloud Run) with a `redirect_uri_mismatch` error. The error occurs because:

1. **OAuth Redirect URI Mismatch**: Google OAuth validates redirect URIs against the registered authorized redirect URIs in the Google Cloud OAuth consent screen. When deployed to Cloud Run, the app is served from a Cloud Run URL, but the Firebase config uses `authDomain: "cohere-flow.firebaseapp.com"`.

2. **Firebase Auth Handler Route**: Firebase uses the `__/auth/handler` route to process OAuth callbacks. This route must be accessible and not intercepted by the service worker or other middleware.

3. **Cross-Origin Storage Access**: Modern browsers block third-party storage access, which can prevent Firebase auth popups from working correctly when the app is not hosted on Firebase Hosting.

## Root Cause

When using `signInWithPopup()` or `signInWithRedirect()`, Firebase Authentication:
1. Opens a popup/redirect to `https://<authDomain>/__/auth/handler` (default: `https://<project>.firebaseapp.com/__/auth/handler`)
2. Google OAuth redirects back to this handler URL after authentication
3. Firebase processes the callback and closes the popup/redirects back to the app

In Firebase App Hosting (Cloud Run):
- The app is served from a Cloud Run URL (e.g., `https://cohere-flow-xxxxx-uc.a.run.app`)
- But the `authDomain` in Firebase config is `cohere-flow.firebaseapp.com`
- Google OAuth validates that the redirect URI matches registered URIs
- If `https://cohere-flow.firebaseapp.com/__/auth/handler` is not in the authorized redirect URIs, the sign-in fails

## Solution

### Fix 1: Update Firebase Client Configuration (✅ Applied)

**File**: `lib/firebase/client.ts`

The Firebase JS SDK automatically checks for the `FIREBASE_WEBAPP_CONFIG` environment variable during build (provided by Firebase App Hosting). This ensures the correct `authDomain` is used.

```typescript
// Firebase App Hosting provides FIREBASE_WEBAPP_CONFIG during build
// which allows automatic SDK initialization without explicit config
// This ensures proper authDomain configuration for Cloud Run deployments
const firebaseConfig = (() => {
  // Check for Firebase App Hosting auto-configuration (build-time)
  if (typeof process !== 'undefined' && process.env?.FIREBASE_WEBAPP_CONFIG) {
    try {
      return JSON.parse(process.env.FIREBASE_WEBAPP_CONFIG);
    } catch {
      // Fall through to manual config
    }
  }
  
  // Fallback to manual configuration for local development
  return {
    apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
    authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
    projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
    storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
    messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
    appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
  };
})();
```

### Fix 2: Configure Google Cloud OAuth Consent Screen (⚠️ Manual Step Required)

This must be done in the [Google Cloud Console](https://console.cloud.google.com/apis/credentials/consent):

1. Go to **APIs & Services** → **OAuth consent screen**
2. Select your app (or create one)
3. Go to **Credentials** → **OAuth 2.0 Client IDs**
4. Edit the Web client ID
5. Add the following **Authorized redirect URIs**:

```
https://cohere-flow.firebaseapp.com/__/auth/handler
https://cohere-flow.web.app/__/auth/handler
```

6. Also add these as **Authorized JavaScript origins**:

```
https://cohere-flow.firebaseapp.com
https://cohere-flow.web.app
```

### Fix 3: Update Firebase App Hosting Configuration (✅ Applied)

**File**: `apphosting.yaml`

Added proper resource configuration and ensured environment variables are available during build:

```yaml
runConfig:
  runtime: nodejs22
  cpu: 1
  memoryMiB: 1024
  minInstances: 0
  maxInstances: 10
  concurrency: 80
```

### Fix 4: Service Worker Configuration (⚠️ May Need Update)

**File**: `public/sw.js` (auto-generated by workbox)

The service worker must NOT intercept the `__/auth` routes. If using a custom service worker configuration (e.g., with `vite-plugin-pwa` or `workbox`), add:

```javascript
// Allow Firebase auth routes to pass through service worker
registerRoute(
  ({url}) => url.pathname.startsWith('/__/auth'),
  new NetworkFirst()
);
```

Or in Vite PWA config:

```javascript
// vite.config.ts
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    VitePWA({
      workbox: {
        navigateFallbackDenylist: [/__/], // Allows firebase auth routes to pass through
      },
    }),
  ],
})
```

## Additional Notes

### Why This Works

1. **Firebase App Hosting Auto-configuration**: The `FIREBASE_WEBAPP_CONFIG` environment variable is automatically populated by Firebase App Hosting during build. It contains the correct Firebase config including the proper `authDomain` for the deployment.

2. **OAuth Redirect URIs**: By registering `https://cohere-flow.firebaseapp.com/__/auth/handler` as an authorized redirect URI, Google OAuth knows it's valid for this Firebase project, regardless of where the app is actually hosted.

3. **Service Worker**: Firebase auth uses popup windows and redirect flows that must not be intercepted by service workers. The `__/auth` routes are Firebase's internal auth handler and must be allowed to pass through.

### Troubleshooting

If Google sign-in still fails after applying these fixes:

1. **Check Google Cloud OAuth Configuration**:
   - Verify the authorized redirect URIs are correct
   - Ensure the OAuth consent screen is published (not in testing mode, or add test users)

2. **Check Browser Console**:
   - Look for specific error messages
   - Common errors:
     - `redirect_uri_mismatch`: OAuth configuration issue
     - `popup_closed_by_user`: Service worker blocking popup
     - `auth/popup-blocked`: Browser blocked popup

3. **Test in Incognito Mode**:
   - Disable browser extensions that might block popups
   - Test without cached service worker

4. **Check Firebase Project Settings**:
   - Ensure Google sign-in is enabled in Firebase Console → Authentication → Sign-in method
   - Verify the authorized domains include `cohere-flow.firebaseapp.com`

5. **Review Service Worker**:
   - Unregister service worker temporarily to test
   - Check if service worker is intercepting auth routes
