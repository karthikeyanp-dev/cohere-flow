## Firestore Data Model

```
/users/{uid}
  uid, email, displayName, photoURL
  role: "admin" | "member"
  createdAt: Timestamp

/projects/{projectId}
  name, description
  status: "active" | "archived"
  createdBy: uid
  memberIds: uid[]              ← denormalized for security rules
  workflow: [{ id, label, color, order }]
  createdAt, updatedAt: Timestamp

/projects/{projectId}/tasks/{taskId}
  title, description
  status: string                ← matches workflow stage id
  assigneeId: uid | null
  creatorId: uid
  projectId: string             ← for composite indexes
  createdAt, updatedAt: Timestamp

/notifications/{notifId}
  recipientId: uid
  type: "task_assigned"
  taskId, projectId, taskTitle, actorName: string
  read: boolean
  createdAt: Timestamp
```

**Composite Indexes:**
- `tasks`: `(projectId ASC, status ASC, createdAt DESC)`
- `tasks`: `(assigneeId ASC, createdAt DESC)`
- `notifications`: `(recipientId ASC, read ASC, createdAt DESC)`

## Firestore Security Rules

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /users/{uid} {
      allow read: if request.auth != null &&
        (request.auth.uid == uid || isAdmin());
      allow write: if request.auth.uid == uid;
    }

    match /projects/{projectId} {
      allow read: if isMember(projectId);
      allow create: if isAdmin();
      allow update, delete: if isAdmin();

      match /tasks/{taskId} {
        allow read, write: if isMember(projectId);
      }
    }

    match /notifications/{notifId} {
      allow read, update: if
        request.auth.uid == resource.data.recipientId;
      allow create: if false; // Admin SDK only
    }

    function isAdmin() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid))
             .data.role == 'admin';
    }
    function isMember(projectId) {
      return request.auth.uid in
        get(/databases/$(database)/documents/projects/$(projectId))
        .data.memberIds;
    }
  }
}
```

### Phase 3 — Authentication

**AuthContext** (`contexts/AuthContext.tsx`)
- `onAuthStateChanged` → fetch `/users/{uid}` for role
- Expose: `user`, `role`, `loading`, `signIn`, `signInWithGoogle`, `signOut`, `resetPassword`

**Pages:**
- `/login` — Email form + "Continue with Google" button
- `/register` — Creates `/users/{uid}` with `role: "member"` on first sign-up; Google users auto-created on first OAuth login
- `/reset-password` — `sendPasswordResetEmail`

**Middleware** (`middleware.ts`)
- Protect all `/(dashboard)` routes; redirect to `/login`
- Block `/admin/*` for non-admins → redirect to `/dashboard`

### Phase 4 — Layout & Navigation

**`app/(dashboard)/layout.tsx`**
- Collapsible sidebar (desktop), bottom nav (mobile)
- Topbar: app name, `NotificationBell`, user avatar dropdown
- Role-aware nav items (Admin section hidden for members)

**`components/layout/EmptyState.tsx`**
- Shown on `/dashboard` when `userProjects.length === 0`
- Message: *"You haven't been added to any projects yet. Ask your admin to invite you."*
- No action buttons visible for members; Admin sees "Create your first project" CTA

### Phase 5 — Project Management

**Member view** (`/projects`) — lists only projects where `memberIds` contains current UID via Firestore query  
**Admin view** (`/admin/projects`) — full CRUD with `ProjectForm` (name, description, initial workflow)  
**`useProjects` hook** — Firestore `onSnapshot` filtered by `memberIds array-contains uid`  

Actions:
- `createProject(data)` — Admin only; initializes with default workflow `["To-Do", "In Progress", "In Review", "Done"]`
- `updateProject(id, data)` — Admin only
- `archiveProject(id)` — sets `status: "archived"`; archived projects hidden from members

---

### Phase 6 — Workflow Configuration

**`/projects/[projectId]/settings`** (Admin only)
- Drag-and-drop stage list using `@dnd-kit/sortable`
- Per-stage: label input, color picker (8 preset colors), reorder handles, delete button
- Validation: minimum 2 stages, labels must be unique, no blank labels
- On save: patches `projects/{projectId}.workflow`; warns if existing tasks use a deleted stage

**`WorkflowEditor` component** — sortable list with add/remove/rename inline

---

### Phase 7 — Task Management & Kanban Board

**`/projects/[projectId]/board`** — Real-time Kanban
- `KanbanBoard` renders columns from `project.workflow` in order
- Each `Column` shows stage label, color accent, task count, scrollable `TaskCard` list
- Drag-and-drop cards between columns (`@dnd-kit`) → `updateTask({ status: newStageId })`
- "Add Task" button per column opens `TaskForm` pre-filled with that stage

**`TaskCard`** — title, assignee avatar chip, creation date; click → `TaskModal`

**`TaskModal`** — full detail view:
- Inline editable title + description
- Status dropdown (workflow stages only)
- Assignee picker (project members only)
- Creator + timestamps
- Delete button (soft delete for members; hard delete for Admin)

**`useTasks` hook**
- `createTask` → write to Firestore + POST `/api/notifications/create` if assigneeId set
- `updateTask` → partial update; if `assigneeId` changed → POST notification API
- Real-time: `onSnapshot` on `projects/{projectId}/tasks`

**`/api/notifications/create/route.ts`** (Server Route Handler)
- Validates caller's Firebase ID token server-side
- Uses Admin SDK to write to `/notifications/{id}` (bypasses client rules)

---

### Phase 8 — Admin: Users & Members

**`/admin/users`**
- Lists all `/users` docs
- Toggle role: calls `/api/admin/set-role` → Admin SDK sets Firestore `role` field
- Search/filter by name or email

**`/api/admin/set-role/route.ts`**
- Verifies caller is admin via decoded ID token
- Updates `users/{uid}.role` in Firestore

**`MemberPicker` component**
- Multi-select from `/users` collection
- Used in `ProjectForm` (initial members) and project settings (add/remove members)
- Updates `projects/{projectId}.memberIds` array

### Phase 9 — Notifications System

- Global `onSnapshot`: `notifications` where `recipientId == uid && read == false`

**`NotificationBell`** (in Topbar)
- Indigo badge with unread count (hidden at 0)
- Dropdown: last 5 notifications with task title, actor name, relative time
- "Mark all read" button
- "View all" → `/notifications`
- Bell icon animates (shake) when new unread arrives

**`/notifications` page**

- Grouped by date
- Click → navigate to `/projects/[projectId]/board` and open task modal
